# Spec-Driven Development Rules

These rules define the mandatory process for all development activities in Project Lotus2026.

## Core Process
All development tasks MUST follow these four sequential steps. No code implementation is allowed until Step 4 has been completed and approved.

1.  **Step 1: Vision & Requirements (01-vision)**
    *   Define the ideation, problem statement, and Product Requirement Document (PRD).
    *   **MANDATORY**: Ask for user approval before moving to Step 2.

2.  **Step 2: Design (02-design)**
    *   Define UI/UX components, layout, and Voice-UI (VUI) interactions.
    *   **MANDATORY**: Ask for user approval before moving to Step 3.

3.  **Step 3: Architecture (03-architecture)**
    *   Define system components, API contracts, and database schema.
    *   **MANDATORY**: Ask for user approval before moving to Step 4.

4.  **Step 4: Feature Specification (04-features)**
    *   Define the specific implementation logic for the module/feature.
    *   **MANDATORY**: Ask for user approval before proceeding to implementation.

## Implementation Phase
*   Only after Step 4 is approved, proceed with the actual code implementation.
*   Ensure all code matches the approved specifications in [specs](file:///Users/handonn/Workplace/Lotus2026/.trae/specs).
