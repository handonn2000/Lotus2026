# Coding Style Rules

Standardized coding style rules for the Lotus2026 project to ensure a unified codebase.

## Language Standards
- **TypeScript**: Use TypeScript for all frontend and backend code (where possible).
- **ES6+**: Utilize modern JavaScript features (arrow functions, destructuring, template literals).

## Code Formatting
- **Prettier**: Use Prettier with default project settings for all files.
- **Indentation**: 2 spaces for all file types.
- **Semicolons**: Always use semicolons at the end of statements.
- **Quotes**: Use single quotes for strings, unless double quotes are necessary.

## Naming Conventions
- **Variables & Functions**: `camelCase`.
- **Classes & Components**: `PascalCase`.
- **Constants**: `UPPER_SNAKE_CASE`.
- **Files & Folders**: `kebab-case`.

## Best Practices
- **Strict Types**: Avoid using `any`; define interfaces and types for all data structures.
- **Early Returns**: Use early returns to reduce nesting and improve readability.
- **Pure Functions**: Favor pure functions to make code more predictable and testable.
